# WORLD : TERMINUS - CONTEXT (읽기 전용)

이 문서는 **세계관/작품 맥락 설명용**입니다.  
**구현 지시/자동화 규칙은 절대 여기 쓰지 말고**, `SPEC.md` / `TASKS.md` / `CONSTRAINTS.md`에만 작성합니다.

## 이 작품의 형태
- **세계관 / 소설 / 게임** 3가지를 모두 포함합니다.
- “항목(노션 페이지 1개)”은 **여러 도메인에 동시에 속할 수 있음**을 기본 전제로 합니다.

## 운영 목표(사용자 경험)
- 편집/정리는 **Notion DB(“총정리”)**에서 한다.
- 탐색/연결/전체 조망은 **그래프(마인드맵) 뷰어**로 한다.
- 뷰어는 **노션 웹에서 바로 볼 수 있어야** 한다(노션 내부 실행 불가 → 외부 뷰어 임베드).

## 레이어(의미 체계)
- `CORE` : 세계의 핵/메타/기준점
- `SYSTEM` : 규칙/메커니즘/제약(특히 게임 시스템 포함)
- `NARRATIVE` : 사건/에피소드/플롯
- `PLAYER` : 플레이어에게 노출되는 정보(스포일러/정보량 조절)

## Notion에서 쓰는 핵심 속성(개념 설명)
아래 속성명은 **정확한 한글 표기 그대로** 사용합니다(구현/자동화는 `SPEC.md`에서 규정).
- `도메인` : 세계관/소설/게임 (복수 선택)
- `대분류` : 설정/인물/세력/장소… 같은 큰 분류
- `연관` : 태그(자동 분류/검색 보조)
- `관련 노드` : 항목 간 직접 연결(관계 그래프 엣지)
- `상태` : 초안/정리중/확정

## “하이브리드 맵”이란
- **Notion DB = 원본(SSOT)**  
- **웹 그래프 = 조망/탐색 UI**  
- 최종 목표는 “항목을 만들고 연결만 하면”, 나머지는 자동으로 최신화되어 **노션 안에서 그래프로 편하게 보는 것**입니다.
